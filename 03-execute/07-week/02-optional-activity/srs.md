# SRS (Software Requirements Specification)

## ¿Qué es SRS?
SRS significa **Software Requirements Specification** (Especificación de Requisitos de Software).  
Es un documento que describe de manera detallada **qué debe hacer un sistema de software** y bajo qué condiciones debe operar.

---

## Objetivo del SRS
- Servir como **guía principal** para desarrolladores, diseñadores y testers.
- Definir claramente los **requisitos funcionales y no funcionales**.
- Evitar malentendidos entre clientes y equipo técnico.

---

## Componentes principales

### 1. Introducción
- Propósito del sistema
- Alcance del proyecto
- Definiciones y acrónimos

### 2. Descripción general
- Perspectiva del producto
- Funciones principales
- Tipos de usuarios
- Restricciones

### 3. Requisitos específicos

#### Requisitos funcionales
Describen **qué debe hacer el sistema**.
Ejemplo:
- El sistema debe permitir registro de usuarios.
- El sistema debe procesar pagos.

#### Requisitos no funcionales
Describen **cómo debe comportarse el sistema**.
Ejemplo:
- Rendimiento
- Seguridad
- Usabilidad
- Escalabilidad

---

## Características de un buen SRS
- Claro y preciso
- Completo
- Consistente
- Verificable
- Modificable

---

## Importancia
- Reduce errores en el desarrollo
- Mejora la comunicación
- Facilita pruebas y validación
- Sirve como base para mantenimiento futuro

---
